# Frontend Guardrails

반복적으로 발생했거나 발생 가능성이 높은 UI 구현 문제에 대한 가드레일입니다.

> 지나치게 구체적인 규칙보다 **판단 기준**을 제공하는 것이 목적입니다.
> 같은 실수가 두 번 이상 발생하거나, 구현 후 뒤늦게 발견된 제약은 이 문서에 추가합니다.

---

## 1. 레이아웃 오버플로우

### 문제 패턴
중첩 카드/컨테이너 레이아웃에서 width constraint가 없으면, 콘텐츠가 예상 너비를 초과할 때 부모 경계를 벗어납니다. 특히 다음 상황에서 빈번합니다:

- 사용자가 직접 입력하는 동적 콘텐츠 (긴 텍스트, 한국어 등 멀티바이트)
- 인라인 편집 input이 콘텐츠 크기에 따라 확장되는 경우
- flex/grid 자식 컨테이너에 `min-width: auto` 기본값이 적용되는 경우

### 판단 기준
컨테이너를 만들 때 다음 질문을 거칩니다:

- **"이 컨테이너는 부모를 채워야 하는가?"** → `w-full` 명시
- **"콘텐츠가 넘칠 경우 어떻게 되어야 하는가?"** → `overflow-hidden` 또는 `overflow-auto` 결정
- **"이 콘텐츠가 잘려도 기능적으로 문제없는가?"** → 잘리면 안 되면 `overflow-hidden` 대신 레이아웃(포지셔닝, 높이) 자체를 수정
- **"이 안에 또 다른 컨테이너가 중첩되는가?"** → 각 레벨마다 위 질문들 반복

### 기본 적용 원칙
- 부모를 채워야 하는 자식 컨테이너: `w-full` 기본 적용
- 클리핑이 필요한 컨테이너: `overflow-hidden` 기본 적용
- 명시적 크기가 필요한 경우에만 예외

### 검증 기준
빌드 성공은 레이아웃 정확성을 보장하지 않습니다. 신규 컨테이너 컴포넌트 구현 후:

- [ ] 30자 이상 텍스트(특히 한국어)로 테스트
- [ ] 인라인 편집 상태에서도 레이아웃 유지 확인
- [ ] 중첩 컨테이너 각 레벨에서 경계 확인
