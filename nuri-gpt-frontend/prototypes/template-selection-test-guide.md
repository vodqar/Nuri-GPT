# 일지작성 파이프라인 베타테스트 가이드

## 뷰 구성 및 흐름

```
[템플릿 선택] → [새 템플릿 생성] ─┐
                                 ↓
                              [작성 화면] ──→ [결과 확인]
                                 ↑                    │
                                 └────────────────────┘ (재생성 시)
```

---

## 1. 템플릿 선택 뷰

### 작동 기능
- **템플릿 선택**: 카드 클릭 → 선택 표시(체크 아이콘), 하단 버튼 텍스트 업데이트
- **시작 버튼**: 선택된 템플릿으로 작성 화면 진입
- **빈 목록 처리**: "아직 템플릿이 없습니다" 안내 + 새 템플릿 버튼 강조

### 관리 모드 (관리 버튼 클릭 시)
| 기능 | 동작 |
|------|------|
| 순서 변경 | 드래그 핸들로 드래그앤드롭 |
| 이름 수정 | 템플릿명 클릭 → 인라인 편집 |
| 삭제 | 휴지통 버튼 → 확인 모달 |
| 완료/취소 | 완료 버튼(저장), 뒤로 버튼(변경사항 폐기 확인) |

### 새 템플릿 생성 버튼
- 클릭 시 생성 뷰로 전환
- 생성 완료 시 목록 최상단 추가 + 자동 선택

---

## 2. 새 템플릿 생성 뷰

### 입력 항목
| 항목 | 필수 | 동작 |
|------|------|------|
| 템플릿 이름 | O | 중복 허용, 공백만 제한 |
| 이미지 업로드 | O | 클릭/드래그앤드롭, 파일명/크기 표시, 삭제 가능 |

### 제출 조건
- 이름 입력 + 파일 선택 시 버튼 활성화
- 제출 중 로딩 상태 표시 ("템플릿 분석 및 생성 중...")

### 오류 처리
- 미입력 제출 시: 에러 얼럿 표시
- 생성 중 뒤로가기: 진행 취소 확인 모달

---

## 3. 작성 화면 (Writing View)

### 공통 요소
- **연령 선택**: 0~5세 드롭다운, 미선택 시 제출 불가 + 에러 표시(빨간 테두리)
- **뒤로가기**: 선택 뷰로 복귀, 입력값 유지 여부 확인 필요

### 입력 모드

#### 직접 입력 모드
- **폼 구조**: `semantic_json` 기반 평면화된 필드 목록
- **경로 표시**: breadcrumb 스타일 (예: 놀이 › 실내놀이 › 놀이상황)
- **OCR 버튼**: 각 필드 우측 상단, 클릭 시 이미지 업로드 → OCR 텍스트 삽입
- **입력 저장**: onChange 시 `manualInputs` 객체에 저장 (key: dot notation 경로)

#### 자유 입력 모드
- **단일 textarea**: 전체 내용 한 번에 입력
- **OCR 버튼**: textarea 상단

### 제출 처리
1. 연령 미선택 → 에러 표시 + 해당 위치 스크롤
2. 직접 입력 모드 + 빈 필드 존재 → 확인 모달 ("계속 진행하시겠습니까?")
3. 통과 → 결과 확인 뷰로 전환 + 로딩 오버레이

---

## 4. 결과 확인 뷰 (Result View)

### 구성
- **버전 히스토리**: 드롭다운으로 버전 선택 (v1, v2...)
- **결과 카드**: 필드별 생성된 내용 표시
- **코멘트 기능**: 카드 클릭 또는 수정 버튼 → 코멘트 입력창 토글
- **복사 기능**: 개별 복사(카드별 버튼), 전체 복사(상단 버튼)

### 코멘트 흐름
1. 코멘트 입력 → 확인
2. 해당 카드에 코멘트 뱃지 표시
3. 하단 "반영하여 다시 만들기" 버튼 활성화 (카운트 표시)

### 재생성
- **조건**: 1개 이상 코멘트 존재
- **동작**: 로딩 오버레이 → 새 버전 생성 (기존 버전 유지) → 최신 버전 표시
- **코멘트 반영**: 이전 버전의 코멘트가 새 버전 내용에 반영됨 (시뮬레이션)

### 버전 전환
- 드롭다운 변경 → 해당 버전 내용 표시
- 현재 버전의 코멘트는 유지 (버전별 독립)

## 데이터 구조

### 템플릿
```javascript
{
  id: number,
  name: string,
  createdAt: string,
  badge: string|null,
  semantic_json: Object  // 폼 구조 정의
}
```

### 입력값 (직접 입력 모드)
```javascript
manualInputs: {
  "보육일지.놀이.활동.내용": "입력된 텍스트",
  ...
}
```

### 버전 데이터
```javascript
versions: [
  {
    content: { flatKey: "생성된 내용" },
    comments: { flatKey: "코멘트 내용" }
  }
]
```

## 테스트 체크리스트

### 기본 흐름
- [ ] 템플릿 선택 → 작성 → 완료 → 결과 확인
- [ ] 빈 필드로 제출 시 확인 모달
- [ ] 연령 미선택 시 에러 표시 및 스크롤

### 관리 모드
- [ ] 순서 변경 후 저장/취소
- [ ] 이름 수정 (Enter/Esc/Blur 동작)
- [ ] 삭제 확인 및 애니메이션

### 새 템플릿
- [ ] 이미지 드래그앤드롭 업로드
- [ ] 생성 완료 시 목록에 추가 + 선택

### 결과 확인
- [ ] 코멘트 입력 → 카드 표시
- [ ] 재생성 버튼 활성화/비활성화
- [ ] 버전 전환 시 데이터 유지
- [ ] 개별/전체 복사 및 토스트 알림

### 모드 전환
- [ ] 직접↔자유 모드 전환 시 입력값 유지
- [ ] 작성 화면 ↔ 결과 화면 이동 시 상태 유지
